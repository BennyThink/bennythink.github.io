安装+破解朝鲜Red Star红星root权限

今天看Linux中国，听说隔壁邻居朝鲜出了某自主研发操作系统RedStarOS，根据Fedora19改的哦。听说”可管可控”的，那没有root权限怎么行？

果断下载某网友修改过的安装界面是英文的镜像，果断打开虚拟机。

PS：这家伙果真会在压缩包里动手脚啊，果真是每个文件都给“指纹”了。

 

安装之前吐槽一句，Linux内核是GPL许可证的，按照GPL的传染性，请大家大声告诉我，这个红星开源了没？

 

取个好听的名字，金胖儿。

朝鲜人民穷，我就送它个4G大内存，反正我12G基本上都是空着。

这界面挺现代的啊。

管他啥Boot，/的，直接下一步

设置个用户名密码，我向你保证，你的用户名没在sudoers里。

设置网络，由于我设置的桥接，就直接DHCP了。

点击安装开始……

妈了个鸡的竟然要23分钟，虚拟机硬盘性能差我知道，那也用不了这么长时间啊！

15:43开始安装，现在是16:24，40分钟过去了……

你这速度是有多慢……都给我的硬盘里写了什么鬼东西。有40分钟我都虚拟机装好Win10了

看我这磁盘I/O

打字都一卡一卡的，后悔没把虚拟机位置设置到F盘……

16:38，终于装完了……

头像是只大猩猩，你们知道我是在嘲讽胖胖的。

看不懂朝鲜语，果断输密码回车。

我是真的看不懂啊……根据网友说明，改成英文版的把！

选好了我的Ubuntu镜像

改启动设备（安装完VMWare会问你移除IDE:0设备不？你要是移除了，就重新添加下CD/DVD设备吧）

进来liveCD了

看， 这是在live cd里看到的红星硬盘。

/etc/sysconfig，找到i18n文件， LANG=”ko_KP.UTF-8”,把ko_KP

换成en_US

去/usr/share/config,

去找kdeglobals，同样用文本编辑器打开，先找Country=ko_KP

把ko_KP换成en_IS，最后找Language=ko_KP:en_US,继续en_US

别着急重启，没root权限干毛啊。记得刚开始提到的sudoers不？俺们去手动编辑下那个文件。

这两行的注释都取消。

改过sudoers的人可能知道，直接在root ALL=(ALL:ALL) ALL下添加类似jack ALL=(ALL:ALL) ALL就可以了，也顺手添加下吧。但是鬼知道这个改了什么呢？

Windows下的管理员组叫adminstrators，linux下的就叫轮子了，也就是wheel，只有属于这个组的用户才能使用su命令获得root权限，这个组通常为wheel。

多做点事总是没错的啊。

打开/etc/group，把root,B复制到wheel下，其实就是用户名的缩写。

启动中

我爱你，英文。

我的笔记本分辨率太低了，截图不完整。不过英文超熟悉有木有！

这浏览器不错啊，光明网的首页么？

先把火狐改成英文的

然后回车，英文了。

终端是我们的最爱啊！

这This Mac是什么？传说胖胖是果粉哎。

还想啥啊，sudo passwd root改密码啊。

来看看来看看这都啥年代了啊！还2.6呢。给大家补充下，2.6大概是2003年发布的。

直接su切换到root可能不太好用，反正我的用户名是wheel组的，sudo shutdown -h now关机

然后做快照，猜猜接下来我要干嘛？

 

做个快照，开机

打开我最爱的终端，sudo rm –rf /   会怎样呢

 

对啊，忘记说了，如果想上网，那就移除iptables规则，禁用了或者直接删掉就可以。

sudo chkconfig iptables off

或者

rm /etc/sysconfig/iptables

哇哇哇朝鲜人民终于看见”墙外”的世界啦

好，万事俱备，那么，开始自杀吧！

sudo rm –rf –-no-preserve-root /

没效果啊！

可能是selinux，关闭了！

sudo vi /etc/selinux/config
SELINUX=disabled

或者临时关闭 setenforce 0

妈的rm –rf /直接重启了，还真是啊。

我表示删不掉，放弃了，就这样吧

好，今天的测评到此为止，我要去背英语单词了……

请高人告诉我金胖儿的邮箱

 

 

总结：只要拿到live CD，轻松获取root权限，不过问题来了，既然已经拿到了live cd，还要着垃圾什么
